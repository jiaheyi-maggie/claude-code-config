• Custom Commands (claude-code-config)
/ideate
Greenfield creative exploration — challenges assumptions, proposes pivots, shapes the MVP. Synthesizes optimal hybrid solutions.
/explore-feature <desc>
Scoped feature exploration within an existing project. Reads codebase, maps against existing architecture/data model/patterns, proposes the best approach. Outputs a feature brief ready for /architect.
/create-prd
Generates full PRD with features, acceptance criteria, risks, launch plan, and explicit out-of-scope items.
/architect
Four-pillar architecture review (components, data, failure modes, security). Generates interactive HTML diagrams. Implementation phases with mandatory review gates.
/prime
Loads project structure, key files, and git state before starting work. Front-loads context.
/tdd
Strict Red-Green-Refactor cycle with test quality checklist. Write the test first, watch it fail, then implement.
/bugfix <desc>
5-phase bug fix: understand → reproduce → diagnose → fix → verify. Tracks attempts in .claude/bugs.md. Max 3 attempts then escalate.
/review-feature
4-pass post-implementation review: bugs, logic, engineering quality, product alignment. End-to-end verification with feature table and stress testing.
/security-audit
Comprehensive OWASP Top 10 security audit with auto-fix capability.
/pre-ship
Final quality gate: 7 gates (build, semantics, edge cases, security, performance, DX, product alignment). All must pass.
/generate-prompt
Expands a rough prompt into structured engineering requirements.
/handover
Saves session state to auto-loading file + persists lessons to memory for next session.
/tbc
Drops a bookmark (session ID + JSONL line count). Pair with /catch-up when you return.
/catch-up
Summarizes everything since last /tbc: messages, files changed, decisions, current state.
/summarize <topic>
Searches past JSONL session files for a topic. Produces summary with decisions, outcomes, open items.
/build-features <list>
Feature orchestrator. Takes a list of features, builds dependency DAG, designs shared contracts, coordinates parallel agents through build-review cycles with quality gates. Topological sort, zero file overlap enforcement, circuit breaker after 3 failures.
/launch-multiagent-team
Decision framework for when/how to launch multi-agent teams. Rules: 3+ independent streams, <20% file overlap, 2-3 agent sweet spot.
• Agentic Substrate Commands
/research
Invokes docs-researcher for rapid documentation research with version-accurate sources. Produces ResearchPack.
/plan
Invokes implementation-planner to create surgical, reversible implementation plans with rollback procedures.
/implement
Invokes code-implementer to execute implementation plans with up to 3 self-correction retries.
/workflow
Full Research → Plan → Implement pipeline in one command. Orchestrated by chief-architect.
/context
Analyzes and optimizes context configuration. Prevents context rot, reduces token usage.
• Custom Agents (claude-code-config)
senior-engineer opus
Principal engineer with deep systems knowledge. Architecture, tech selection, performance, system design. Generates interactive HTML diagrams. Backed by engineering-kb.md knowledge base.
code-reviewer opus
Delegates to /review-feature for 4-pass review. Spawned after big features. Always recommends the RIGHT fix, never shortcuts.
product-manager opus
Owns "what" and "why." Ideation, requirements, PRDs, ICE scoring, scope management. Challenges whether things should be built at all.
frontend-engineer opus
React/Next.js specialist. Component architecture, state management, responsive design, CSS 2026, accessibility, performance. Uses Playwright for visual verification before/after UI fixes.
ux-engineer opus
User flows, interaction patterns, design systems. Spacing/color/typography systems, form/table/modal patterns. Mobile-first. Uses Playwright to screenshot before critiquing existing UI.
tech-writer sonnet
API docs, ADRs, changelogs, onboarding guides, READMEs. Writes for the reader who has zero context and is in a hurry.
qa-engineer opus
Post-implementation QA. Verifies features against requirements, finds and fixes bugs, incorporates user tweaks, iterates until solid. Uses Playwright for UI issues. Delegates to frontend-engineer and ux-engineer for specialized fixes.
ui-mockup opus
Reviews codebase, identifies unbuilt pages/features, generates detailed interactive HTML mockups with all states. Iterates with user, then produces implementation specs for engineers.
pitch-deck opus
Builds polished interactive HTML pitch decks by deeply understanding the product from the codebase. Structures narrative around the moat, presents with proper information hierarchy.
brand-strategist opus
Brand and marketing strategist. Reads the codebase, then writes investor and market-facing materials: pitch decks, one-pagers, exec summaries, landing page copy, press releases, taglines, investor memos. Optimized to attract VCs.
visionary opus
Visionary CTO / product futurist. Deep frontier tech knowledge + bold product thinking. Brainstorming, challenging assumptions, identifying what to build next. Modes: brainstorm, challenge, futures, moonshot, pivot.
ai-engineer opus
Staff AI engineer and industry analyst. Models, agent architectures, MCP, RAG, evals, AI UX, coding tools, competitive landscape. Backed by ai-engineering-kb-2026.md. Always web searches for latest before answering.
